##Настройка keepalived для 2-ух Цодов

#Необходимо настроить noprempt:

* По умолчанию, главный узел имеет высший приоритет. Если он падает, напарник становится мастером. Когда старый сервер оживает, он автоматически возвращает себе роль, отбирая плавающий IP. Это вызывает лишние переключения (прерывания). nopreempt запрещает это. Восстановленный сервер останется запасным, а трафик пойдет через работающего напарника.

````
vrrp_instance VI_1 {
    state BACKUP           # Обязательно для nopreempt!
    interface eth0         # Ваш сетевой интерфейс
    virtual_router_id 51   # Идентификатор кластера
    priority 100           # Высокий приоритет
    advert_int 1
    nopreempt              # Запрет забирать роль
````

#Необходимо настроить unicast:

* Для того, чтобы серверы общались друг с другом напрямую, отправляя данные только по конкретным IP-адресам, а не рассылая их всем подряд.

Для первого сервера
````
    unicast_src_ip 192.168.1.10   # Локальный IP-адрес этого сервера
    unicast_peer {
        192.168.1.11              # Локальный IP-адрес второго (резервного) сервера
    }
````
Для второго сервера
````
    unicast_src_ip 192.168.1.11   # Локальный IP-адрес этого сервера
    unicast_peer {
        192.168.1.10              # Локальный IP-адрес первого сервера
    }
````

#Необходимо настроить advert_int:

* Параметр advert_int задает интервал в секундах, с которым сервер MASTER отправляет сигнальные сообщения (анонсы). Другие серверы (BACKUP) слушают эти сигналы. Если BACKUP не получает сообщение за время, равное трем таким интервалам, он берет работу на себя.
